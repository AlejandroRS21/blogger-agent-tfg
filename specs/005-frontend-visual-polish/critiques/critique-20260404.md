# Critique Report: Frontend Visual Polish & QA

**Date**: 2026-04-04  
**Feature**: `005-frontend-visual-polish`  
**Verdict**: ⚠️ PROCEED WITH UPDATES

## Executive Summary

The specification and implementation plan for the frontend visual polish are strategically sound and align perfectly with the "Static-First" principle of the project constitution. The focus on typography, responsiveness, and metadata is correct for a blog-centric product. However, there is a critical technical risk identified in the implementation of `generateMetadata` and `generateStaticParams` regarding data integrity that must be addressed to prevent build failures in the static export pipeline.

---

## Product Lens Findings

### 3b. User Value Assessment
- **Finding**: The plan correctly identifies that "readability" is the core value.
- **Severity**: ✅
- **Suggestion**: Ensure that the `text-balance` applied in T006 does not negatively impact performance on very long AI-generated posts by using it only on headings if necessary.

### 3d. Edge Cases & User Experience
- **Finding**: AI-generated content can often contain deeply nested or malformed HTML tags (e.g., within tables) that `isomorphic-dompurify` might strip or render poorly.
- **Severity**: 💡 Recommendation
- **Suggestion**: Add a specific task to verify the rendering of "worst-case" complex tables generated by the engine.

---

## Engineering Lens Review

### 4b. Failure Mode Analysis
- **Finding**: The `generateStaticParams` crash (`slug undefined`) seen in recent terminal logs is a symptom of data-code mismatch. The current plan addresses it in T003, but it lacks a "validation" step to prevent future regressions if the Python pipeline outputs invalid JSON.
- **Severity**: 🎯 Must-Address
- **Suggestion**: Add a task (or enhance T002) to create a simple validation script (or zod schema) that the frontend build can use to fail-fast if a post is missing its slug.

### 4d. Performance & Scalability
- **Finding**: Large SEO metatags and OG images can increase the static HTML size significantly if not handled properly.
- **Severity**: 💡 Recommendation
- **Suggestion**: Ensure OG images are referenced via absolute URLs (required by most social platforms) and investigate if a default "OG Image Generator" task should be added to the Python backend in a future feature.

### 4f. Operational Readiness
- **Finding**: The `output: export` mode in Next.js is sensitive to any dynamic `searchParams` or server-side logic.
- **Severity**: 🎯 Must-Address
- **Suggestion**: Add a check to T012 to specifically verify that no ahrefs or links point to `/api/*` or other non-static routes, as these will 404 in the live environment.

---

## Cross-Lens Synthesis

The intersection of **Data Integrity (Engineering)** and **Trustworthy Presentation (Product)** is the highest priority. If the `slug` is missing, the page doesn't build; if the metadata is broken, the link looks like spam. Addressing the "slug undefined" error is the absolute prerequisite for all other visual polish tasks.
